📋 第一章

安全演进：历史风险与现状

从早期高危漏洞，到 v3.12 的安全加固，再到当前仍未解决的威胁——了解龙虾系统的完整安全轨迹。

01

🔥 远程代码执行漏洞（高危）

CVE-2026-25253，CVSS 8.8。攻击者构造恶意 URL 诱使用户点击，通过跨站 WebSocket 劫持与认证令牌泄露组合攻击，劫持会话并获得 operator.admin 权限。即使系统仅限本地访问，通过浏览器打开恶意页面仍存在远程利用风险。

02

☠️ 供应链投毒风险

官方技能市场 ClawHub 因缺乏有效的插件签名验证与运行时沙箱隔离，已成为恶意软件（如远程访问木马）的传播渠道。

03

⬆️ 权限提升与横向移动

系统默认被授予操作系统级权限，一旦被攻陷，极易成为攻击者在内部网络横向渗透的跳板，严重违背最小权限原则。

04

💉 提示词注入攻击

利用大语言模型的语义歧义，通过在网页、PDF 等文件中嵌入隐藏文本（白色字体、零宽字符），诱导系统误判并执行恶意指令，属于非传统攻击面。

05

🌐 默认配置脆弱性

系统的默认配置网络暴露面过大，极易被自动化扫描工具发现并利用，属于低门槛高收益的攻击目标。

v3.12 重要里程碑：以下四项关键漏洞在 3.12 版本中得到修复，系统整体安全性显著提升。
✓

🔑 设备配对机制升级

将长期有效的共享网关凭证嵌入配对码模式，改为仅用于初始引导的短时令牌，大幅降低凭证泄露带来的永久性接管风险。

✓

✅ 命令执行审批流程修复

修复了通过特定调用方式绕过命令执行审批流程的问题，防止了未授权任意命令的执行。

✓

🧹 零宽字符注入防护

增强输入清洗机制，有效阻断利用 Unicode 零宽字符在审批提示中隐藏恶意指令的攻击路径。

✓

🛡️ SSRF 与路径遍历加固

强化文件工具的路径检查，禁止路径逃逸至系统目录；同时阻止对云环境元数据地址等内网特殊地址的请求。

仍存在
🎭

间接提示词注入

通过外部内容（邮件、网页）向 Agent 注入隐藏指令，诱导其利用持有权限执行攻击行为。

仍存在
🔌

云集成插件新漏洞

如 Nextcloud Talk 等云集成插件中持续出现新的安全漏洞，需持续跟踪修复。

仍存在
🛒

ClawHub 供应链风险

技能市场缺乏完善的签名验证机制，恶意插件仍可能流入生态，威胁所有用户。

仍存在
⚙️

默认配置攻击面

默认配置不当导致的网络暴露问题尚未根本性解决，需要手动收敛。

⚠️ 特别关注
🤖

代理权漂移（Agency Drift）

龙虾作为"自主智能体"，核心风险在于意图与执行的脱节。授予其聊天工具、文件系统或云端权限时，你实际赋予了它一个"动态身份"——而 LLM 本质是概率语义预测，非确定性逻辑，可能在以下场景失控：

🌀 语义偏移

处理复杂指令时，"清理临时文件"可能演变为"格式化关键数据"。

⚡ 越权滥用

利用"合法" API 令牌，在未经审核情况下执行高危动作（如群发敏感财报）。

🕵️ 受困代理攻击

攻击者通过含隐藏指令的邮件，诱导 Agent 用其持有权限攻击内部系统。

🛡️ 第二章

构建安全的养殖环境与管控体系

系统本身安全能力有限，必须依赖外部环境与管控措施。以下三层防线构成完整的安全管理体系。

2.1  筑牢"养殖"环境安全基线
📦
隔离部署

部署隔离

将系统部署于 Docker 容器、虚拟机或专用云主机等隔离环境，严格限制可访问的文件路径。严禁使用特权帐号。

🌐
网络防护

网络代理与隔离

避免实例间直接通信，通过代理中转。部署于防火墙/IDS/IPS 之后，通过白名单对网络访问进行审计和审批。

🔍
技能管理

技能安全管理

所有待安装技能须经严格安全审查与沙箱验证，技能更新采取手动审批模式。

🔑
密钥保护

凭据管理

使用 Vault 类产品集中管理密码、API 密钥等敏感变量，实施定期轮换策略。

📝
合规审计

日志审计

强制开启并集中收集所有活动日志，定期进行安全审计与分析，确保可追溯性。

🚦
操作管控

操作管控

部署提示词安全检测与过滤功能，对识别出的高危操作强制添加人工确认环节。

⚡ 最小权限原则：对数据和 API 的访问，只需读取就不能给写入权限。权限最小化是降低代理权漂移危害的根本手段。
2.2  实施日常安全监控
🔎

网络层监控

监控防火墙、IPS、代理等设备的告警信息，对异常域名/IP 访问及进出流量实施持续监控。

🖥️

宿主系统监控

实时日志审计、系统完整性监控，对敏感路径的文件读写行为与运行时状态进行监控。

🤖

AI 护栏

监控系统与大模型之间的交互信息，过滤敏感数据输出，抵御提示词注入、越狱等针对 AI 模型的攻击。

📊

资源监控

监控模型 Token 使用量，异常消耗可能是失控行为或攻击的早期信号。

2.3  非受控事件应急预案

一旦发现龙虾出现非受控或恶意行为，立即按以下步骤执行：

1

⚡ 快速隔离

立即隔离其所在的宿主环境，阻断网络连接与进一步影响。速度是第一要务。

2

🔬 取证与分析

保留完整证据链，追溯并定位潜在的被感染节点，评估安全事件造成的损失范围。

3

🕵️ 溯源

可部署蜜罐等手段捕获入侵手段，检查虾池内所有龙虾的整体健康状况。

🏢 第三章

外部解决方案评估

两类外部方案：第三方安全厂商 + 云宿主平台，各有侧重，适合不同部署场景。

3.1 第三方安全厂商
🌿
青藤
企业级一体化方案
  • 集中部署于其管控宿主系统
  • 自研代理集成防火墙、IPS、代理及 AI 护栏
  • 统一控制平台：行为监控 + 私有技能管理
  • 关键数据脱敏与密钥保护（需深度集成）
⚠️ 脱敏功能依赖企业内系统与其平台的集成深度
🛰️
威努特
主机软件形式
  • 以主机软件形式部署，灵活性更高
  • 内置防火墙、IPS、代理及 AI 护栏能力
  • 可通过软件定制预置安全配置
  • 提升易用性，降低运维门槛
✅ 可在无影云电脑部署 WinClaw 进行用户测试
3.2 云宿主平台
☁️
无影云电脑
阿里云 · 完整云主机
  • 完整云主机，用户自行管理
  • 支持内网与外网访问
  • 公司内部可通过客户端或网页登录管理
⚠️ 测试中连接速度较慢，可能与分配带宽有关
🌋
火山引擎
字节跳动 · AI 云平台
  • 支持部署云主机
  • 字节跳动 AI 生态集成优势

📌 第四章

结论与测试建议

三点核心结论，一条近期可行的测试路径。

🏠

环境安全可控

基于现有成熟安全技术和产品，可以构建相对安全的运行环境，有效阻断外部攻击，并检测由内向外泄露的风险。

🎲

本体行为不确定

系统自身行为无法 100% 预测。不同的大语言模型、具有多义性的提示词均可能导致预期外偏差，本体行为安全性仍是未知量。

🔗

模型依赖风险

目前使用的公有大模型服务不承诺信息安全。青藤、威努特在处理企业敏感数据时，均采用私有化部署大模型方案以规避此风险。

🧪 近期可行测试
💡

推荐测试路径

可针对业务需求，在无影云电脑上部署一个或几个外部厂商改造过的 Claw 版本进行用户测试。

在 POC 案例的选择上，建议优先选取仅使用外部数据的 use case，以降低数据泄露风险，例如：

🌍 外部情报收集

抓取公开的行业动态、竞争对手资讯等外部信息，不涉及企业内部敏感数据。

🔍 Open Source Competition Intelligence

基于开源渠道进行竞品分析，全程使用公开数据，风险可控。
